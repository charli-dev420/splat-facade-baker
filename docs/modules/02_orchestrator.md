# Bloc 2 — Local Orchestrator + ComfyUI Background

The orchestrator is the local production backbone for SFB. It does not generate assets by itself; it records projects, assets, workflows, jobs and artifacts, then delegates work to local engines:

```text
SFB Studio / CLI / API
→ SQLite registry
→ job queue
→ local worker
→ Blender / ComfyUI / SFB Core / shell
→ artifact registry
```

## Current status in v2.2

Implemented:

- `sfb-orch` CLI.
- `sfb-api` FastAPI local service.
- SQLite registry.
- project registry.
- asset import from `dataset_manifest`.
- workflow metadata registry.
- job queue.
- artifact registry.
- local `noop` engine.
- local `shell` engine.
- `sfb_bake_maps` engine wrapper.
- `blender_capture` job wrapper.
- ComfyUI HTTP client.
- ComfyUI workflow template injection.
- ComfyUI dry-run jobs.
- capture-plan to job batch creation.
- minimal Studio UI dashboard connected to the local API.

Not implemented yet:

- robust parallel workers.
- full ComfyUI output file download/copy policy.
- advanced retry classes.
- websocket live UI events.
- job cancellation of external subprocesses.
- authentication. This is a local development server.

## Initialize a workspace

```bash
sfb-orch --workspace workspace init \
  --project-id medieval_sfb_v1 \
  --name "Medieval SFB v1"
```

This creates:

```text
workspace/
├─ orchestrator/
│  ├─ sfb_orchestrator.sqlite3
│  └─ logs/
├─ artifacts/
└─ workflows/
```

## Register a Dataset Factory manifest

```bash
sfb-orch --workspace workspace assets register-manifest \
  workspace/manifests/medieval_gold_v0_mv8_split.json \
  --project-id medieval_sfb_v1
```

List assets:

```bash
sfb-orch --workspace workspace assets list --project-id medieval_sfb_v1
```

## Create and run a local test job

```bash
sfb-orch --workspace workspace jobs create \
  --project-id medieval_sfb_v1 \
  --engine noop \
  --param note='"first queue test"'

sfb-orch --workspace workspace jobs run-next \
  --project-id medieval_sfb_v1

sfb-orch --workspace workspace artifacts list \
  --project-id medieval_sfb_v1
```

## Create Blender capture jobs from a capture plan

A capture plan generated by Bloc 1 contains one row per asset/view. The orchestrator groups rows by `asset_id`, so one Blender job renders all selected views for one asset.

```bash
sfb-orch --workspace workspace jobs from-capture-plan \
  workspace/plans/medieval_gold_mv8.jsonl \
  --project-id medieval_sfb_v1 \
  --engine blender_capture \
  --view-contract examples/view_contracts/MV8_OBJECT.json \
  --script tools/render_glb_turntable.py \
  --resolution 1024
```

Run one queued job:

```bash
sfb-orch --workspace workspace jobs run-next --project-id medieval_sfb_v1
```

Run up to 20 jobs:

```bash
sfb-orch --workspace workspace jobs run-all \
  --project-id medieval_sfb_v1 \
  --limit 20
```

Or keep a simple worker alive:

```bash
sfb-orch --workspace workspace worker \
  --project-id medieval_sfb_v1 \
  --poll-interval 2
```

## Register a ComfyUI workflow metadata file

```bash
sfb-orch --workspace workspace workflows register \
  workflows/comfyui/examples/noop_comfy.metadata.json \
  --project-id medieval_sfb_v1
```

Create a ComfyUI dry-run job. This does not contact ComfyUI; it only injects parameters into the workflow template and registers the injected workflow as an artifact.

```bash
sfb-orch --workspace workspace jobs create \
  --project-id medieval_sfb_v1 \
  --engine comfyui \
  --workflow-id noop_comfy_image_v1 \
  --param input_image='"asset_front.png"' \
  --param filename_prefix='"SFB/test"' \
  --param dry_run=true

sfb-orch --workspace workspace jobs run-next --project-id medieval_sfb_v1
```

## Engines

### `noop`

Writes a small job report and completes. Useful for testing the registry and UI.

### `shell`

Runs `params.command`. Example:

```bash
sfb-orch --workspace workspace jobs create \
  --project-id medieval_sfb_v1 \
  --engine shell \
  --param command='["python", "--version"]'
```

### `sfb_bake_maps`

Wraps `sfb_core.cli bake-maps`. Required params:

```json
{
  "albedo": "path/to/albedo.png",
  "alpha": "path/to/alpha.png",
  "depth": "path/to/depth.png",
  "out": "exports/DemoWall"
}
```

Optional params:

```text
name, width_m, height_m, max_depth_m, grid, view_contract, view_id, mobile_tier
```

### `blender_capture`

Wraps `tools/render_glb_turntable.py`. Required params:

```json
{
  "input": "sources/wall.glb",
  "asset_id": "wall",
  "view_contract": "examples/view_contracts/MV8_OBJECT.json",
  "out": "workspace/renders/wall"
}
```

Optional params:

```text
blender, script, resolution, views
```

### `comfyui`

Loads a workflow template, injects params, and submits it to ComfyUI unless `dry_run=true`.

## Design rules

- ComfyUI is a worker, not the source of truth.
- Every important output should become an artifact.
- Jobs must be restartable and inspectable.
- The registry owns metadata; folders only store files.
- This API is local-only for now.
