# Bloc 2 — Local Orchestrator + ComfyUI Background

The orchestrator is the local production backbone for SFB. It does not generate assets by itself; it records projects, assets, workflows, jobs and artifacts, then delegates work to local engines:

```text
SFB Studio / CLI / API
→ SQLite registry
→ job queue
→ local worker
→ Blender / ComfyUI / SFB Core / shell
→ artifact registry
```

## Current status in v2.2

Implemented:

- `sfb-orch` CLI.
- `sfb-api` FastAPI local service.
- SQLite registry.
- project registry.
- asset import from `dataset_manifest`.
- workflow metadata registry.
- job queue.
- atomic SQLite job claiming.
- automatic retry up to `max_attempts`.
- artifact registry.
- local `noop` engine.
- local CLI-only `shell` engine.
- `sfb_bake_maps` engine wrapper.
- `blender_capture` job wrapper.
- ComfyUI HTTP client.
- ComfyUI workflow template injection.
- ComfyUI dry-run jobs.
- capture-plan to job batch creation.
- minimal Studio UI dashboard connected to the local API.

Not implemented yet:

- robust parallel workers.
- full ComfyUI output file download/copy policy.
- advanced retry classes.
- websocket live UI events.
- stale-worker recovery and automatic dead-worker requeue.
- authentication. This is a local development server.

## Initialize a workspace

```bash
sfb-orch --workspace workspace init \
  --project-id medieval_sfb_v1 \
  --name "Medieval SFB v1"
```

This creates:

```text
workspace/
├─ orchestrator/
│  ├─ sfb_orchestrator.sqlite3
│  └─ logs/
├─ artifacts/
└─ workflows/
```

## Register a Dataset Factory manifest

```bash
sfb-orch --workspace workspace assets register-manifest \
  workspace/manifests/medieval_gold_v0_mv8_split.json \
  --project-id medieval_sfb_v1
```

List assets:

```bash
sfb-orch --workspace workspace assets list --project-id medieval_sfb_v1
```

## Create and run a local test job

```bash
sfb-orch --workspace workspace jobs create \
  --project-id medieval_sfb_v1 \
  --engine noop \
  --max-attempts 3 \
  --param note='"first queue test"'

sfb-orch --workspace workspace jobs run-next \
  --project-id medieval_sfb_v1

sfb-orch --workspace workspace artifacts list \
  --project-id medieval_sfb_v1
```

## Create Blender capture jobs from a capture plan

A capture plan generated by Bloc 1 contains one row per asset/view. The orchestrator groups rows by `asset_id`, so one Blender job renders all selected views for one asset.

```bash
sfb-orch --workspace workspace jobs from-capture-plan \
  workspace/plans/medieval_gold_mv8.jsonl \
  --project-id medieval_sfb_v1 \
  --engine blender_capture \
  --view-contract examples/view_contracts/MV8_OBJECT.json \
  --script tools/render_glb_turntable.py \
  --resolution 1024
```

Run one queued job:

```bash
sfb-orch --workspace workspace jobs run-next --project-id medieval_sfb_v1
```

Run up to 20 jobs:

```bash
sfb-orch --workspace workspace jobs run-all \
  --project-id medieval_sfb_v1 \
  --limit 20
```

Or keep a simple worker alive:

```bash
sfb-orch --workspace workspace worker \
  --project-id medieval_sfb_v1 \
  --poll-interval 2
```

## ComfyUI Workflow Gates

ComfyUI examples are not shipped as public product demos until they have a real
operator-owned workflow template and live validation. The repository keeps a
dry-run fixture under tests so the orchestrator injection path stays covered
without presenting a no-op graph as an MVP demo.

```bash
python tools/comfyui_demo_gate.py --workspace workspace/comfyui_demo_gate
```

The gate always runs the orchestrator dry-run fixture. Live ComfyUI validation
requires a reachable server and an operator-provided real metadata file.

## Start the local API

```bash
SFB_WORKSPACE=workspace sfb-api
```

The default API runs at:

```text
http://127.0.0.1:8765
```

Useful endpoints:

```text
GET  /health
POST /api/projects
GET  /api/projects
POST /api/assets/register-manifest
GET  /api/assets
POST /api/workflows/register
GET  /api/workflows
POST /api/jobs
GET  /api/jobs
GET  /api/jobs/{job_id}
POST /api/jobs/{job_id}/retry
POST /api/jobs/run-next
GET  /api/artifacts
GET  /api/comfy/status
GET  /api/file?path=...
```

The HTTP API accepts only `noop`, `comfyui`, `sfb_bake_maps` and `blender_capture`
jobs. Direct `shell` jobs are intentionally rejected over HTTP.
`/api/file` serves only files resolved inside the active workspace or artifact
directory.
Jobs are claimed atomically in SQLite. A failed attempt is requeued while
`attempt < max_attempts`; after the final attempt the job becomes `failed`.
Manual retry is only allowed from terminal/review states and resets `attempt` to
`0`.
Claimed jobs expose `worker_id`, `process_id`, `heartbeat_at`,
`cancel_requested_at` and `cancel_reason`. Cancelling a queued job moves it
directly to `cancelled`. Cancelling a running job moves it to `cancelling`; the
runner then stops its local subprocess best-effort and marks the job
`cancelled` only after the process has ended. ComfyUI cancellation stops the
orchestrator wait loop and attempts ComfyUI `/interrupt`, but the external
ComfyUI server remains responsible for prompts that already completed.

## Engines

### `noop`

Writes a small job report and completes. Useful for testing the registry and UI.

### `shell`

CLI-only local operator engine. It is not accepted by `POST /api/jobs`.
`params.command` must be a JSON argv array; shell command strings are rejected.
Example:

```bash
sfb-orch --workspace workspace jobs create \
  --project-id medieval_sfb_v1 \
  --engine shell \
  --param command='["python", "--version"]'
```

### `sfb_bake_maps`

Wraps `sfb_core.cli bake-maps`. Required params:

```json
{
  "albedo": "path/to/albedo.png",
  "alpha": "path/to/alpha.png",
  "depth": "path/to/depth.png",
  "out": "exports/DemoWall"
}
```

Optional params:

```text
name, width_m, height_m, max_depth_m, grid, view_contract, view_id, mobile_tier
```

Declared `output_paths` are strict: they must be JSON arrays of file paths that
resolve inside the active workspace and exist after a successful process run.
Missing, malformed or out-of-workspace outputs fail the job instead of being
silently ignored.

### `blender_capture`

Wraps `tools/render_glb_turntable.py`. Required params:

```json
{
  "input": "sources/wall.glb",
  "asset_id": "wall",
  "view_contract": "examples/view_contracts/MV8_OBJECT.json",
  "out": "workspace/renders/wall"
}
```

Optional params:

```text
blender, script, resolution, views
```

### `comfyui`

Loads a workflow template, injects params, and submits it to ComfyUI unless `dry_run=true`.

## Design rules

- ComfyUI is a worker, not the source of truth.
- Every important output should become an artifact.
- Jobs must be restartable and inspectable.
- The registry owns metadata; folders only store files.
- This API is local-only for now.
